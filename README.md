# Automated Attendance Management System (AAMS) 

A comprehensive desktop application and system analysis project designed to replace legacy paper-based university attendance tracking with a secure, biometric-enabled digital solution.

## Business Value & ROI
* **The Problem:** Legacy paper-based attendance is inefficient, prone to proxy-attendance (cheating), and incurs high administrative and printing costs.
* **The Solution:** An automated electronic system utilizing biometric verification and encrypted daily access codes.
* **Projected Impact:** Engineered to deliver an estimated annual cost savings of **200,000 EGP**, alongside significant improvements in security, reporting speed, and academic integrity.

##  Tech Stack & Architecture
* **Frontend/GUI:** C# (.NET Windows Forms) featuring dynamic component generation and modular UI panels.
* **Database Design:** Designed for MySQL (Centralized Architecture).
* **System Design:** Comprehensive Data Flow Diagrams (Context, Level 0, Level 1, Level 2) and Business Requirements Documentation.

##  Key Features
* **Biometric Verification:** Validates student identity via fingerprint data simulation.
* **Encrypted Daily Codes:** Generates unique, time-stamped attendance codes per session.
* **Real-time Dashboard:** Interactive UI for professors to monitor attendance and location status.
* **Automated Reporting:** Generates, filters, and exports comprehensive attendance reports by course and date.

##  Repository Structure
* `AttendanceManagementSystem/` - Contains the C# WinForms source code, programmatic UI logic, and event handlers.
* `Automated Attendance Management System.pdf` - The complete System Analysis & Design Document, including ROI calculations, Technical Architecture Decisions, and DFDs.
